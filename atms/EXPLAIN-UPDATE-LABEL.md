# `update-label` 함수 설명

## 한 줄 요약

새로운 환경(environment)들을 노드의 기존 label에 추가하되,
**중복되거나 불필요한 환경을 제거하여 최소성을 유지**합니다.

## 핵심 개념

- **label**: 한 노드가 "참"이 되는 모든 최소 환경의 집합.
  예: `[{A1}, {A2}]` → "A1을 가정하면 참" 또는 "A2를 가정하면 참"
- **환경(env)**: 가정(assumption)들의 집합.

## 동작 과정

### 1단계: 새 환경을 기존 label과 비교

각 새 환경을 기존 label의 모든 환경과 `compare-env`로 비교합니다:

| 비교 결과 | 의미 | 처리 |
|-----------|------|------|
| `:EQ` | 새 환경 = 기존 환경 (동일) | 새 환경 **버림** |
| `:S21` | 새 환경 ⊃ 기존 환경 (상위집합) | 새 환경 **버림** (더 큰 건 불필요) |
| `:S12` | 새 환경 ⊂ 기존 환경 (부분집합) | 기존 환경 **제거** (새 것이 더 작으므로 대체) |
| 그 외 | 서로 비교 불가 | 둘 다 **유지** |

### 2단계: nil 정리

- 버려진 항목(nil)들을 리스트에서 제거
- 살아남은 새 환경에 노드를 역참조로 등록 (`env-nodes`)
- 노드의 label을 최종 갱신

### 3단계: 반환

- 실제로 **새로 추가된 환경들만** 반환
- 호출자(`update`)가 이를 보고 후속 전파 여부를 결정

## 예시

```
기존 label = [{A1}]
새 환경    = [{A1, A2}]

compare-env({A1,A2}, {A1}) → :S21  (새 것이 상위집합)
→ {A1,A2} 버림 (이미 {A1}만으로 충분)
→ label 변화 없음 = [{A1}]
→ 반환: [] (추가된 게 없음 → 전파 안 함)
```

```
기존 label = [{A1, A2}]
새 환경    = [{A1}]

compare-env({A1}, {A1,A2}) → :S12  (새 것이 부분집합)
→ 기존 {A1,A2} 제거, {A1} 추가
→ label = [{A1}]  (더 작은 환경으로 대체!)
→ 반환: [{A1}] → 후속 전파 진행
```

## 전체 흐름에서의 위치

```
justify-node → propagate → weave → update → update-label
                                              ↑ 여기!
```

`weave`가 전건들의 label을 교차곱으로 결합한 뒤,
`update`가 `update-label`을 호출하여 결론 노드의 label을 실제로 갱신합니다.
이때 **"최소 환경만 유지"**하는 것이 이 함수의 핵심 역할입니다.
